# Page Lab 보안 검토 보고서 (Security Review Report)

보안 전문가가 제기한 7가지 이슈에 대해 현재 `PageLab` 소스코드를 기반으로 적용 가능성 및 기능 영향도를 검토한 결과입니다.

## 🎯 총평
**전문가가 지적한 4가지 필수 조치 항목을 포함하여 모든 항목은 현재 구조에서 100% 적용 가능합니다.** 
보안 패치를 적용하더라도 현재 프로그램이 제공하는 핵심 기능(PDF/HWP 변환, 그룹화, 병합, 내보내기 등)에는 **아무런 기능적 제약이나 속도 저하가 발생하지 않습니다.**

---

## 🛑 필수 조치 항목 4가지 검토

### 1. 그룹명·파일명 기반 경로 조작 (Path Traversal)
- **적용 가능 여부:** 🟢 **완벽히 적용 가능**
- **기능 영향도:** 없음
- **조치 방안:** 백엔드(`main.py` 및 `file_manager.py`)에서 사용자가 전달한 파일명, 그룹명을 그대로 사용하지 않고, 파이썬의 `os.path.basename()`과 정규표현식을 통해 특수문자(`../`, `\`, `:` 등)를 제거하는 `safe_filename()` 함수를 도입하면 완벽히 차단됩니다.

### 2. contextMenu.innerHTML XSS 취약점
- **적용 가능 여부:** 🟢 **완벽히 적용 가능**
- **기능 영향도:** 없음
- **조치 방안:** 프론트엔드(`app.js`)에서 사용자 입력값(그룹명, 페이지명 등)을 HTML 렌더링에 사용할 때, 태그(`&`, `<`, `>`, `"`, `'`)를 단순 문자로 변환해주는 `escapeHTML()` 함수를 거치도록 수정하면 인젝션 공격이 원천 차단됩니다.

### 3. 백엔드 API 입력 검증 (Validation)
- **적용 가능 여부:** 🟢 **완벽히 적용 가능**
- **기능 영향도:** 없음
- **조치 방안:** 백엔드(`main.py`)의 `export_data` 함수 진입점에 방어 로직을 추가합니다.
  - `export_type`이 지정된 2개(`single_pdf`, `single_zip`) 중 하나인지 확인
  - `rotation` 값이 0, 90, 180, 270 중 하나인지 확인
  - `file_path`가 실제로 앱이 허용한 임시 폴더(`temp_dir`) 내부에 존재하는 정상적인 PDF 파일인지 검증

### 4. 드래그앤드롭 파일명 정제
- **적용 가능 여부:** 🟢 **완벽히 적용 가능**
- **기능 영향도:** 없음
- **조치 방안:** `upload_dropped_file_bytes` 함수에서 프론트엔드가 보낸 `filename`을 그대로 쓰지 않고, 1번 항목과 동일하게 `safe_filename(os.path.basename(filename))` 처리를 거쳐 안전한 파일명으로만 저장하도록 수정합니다.

## 🔍 추가 권고 항목 3가지 검토

### 5. HWP COM 자동화 신뢰 문제
- **적용 가능 여부:** 🟡 **정책적 적용 (코드 변경 불필요)**
- **조치 방안:** 이 부분은 코드로 막을 수 있는 영역이라기보다는 운영 환경의 문제입니다. `README.md`에 "본 프로그램은 관리자 권한으로 실행하지 마십시오." 및 "출처가 불분명한 악성 HWP 파일은 변환을 시도하지 마십시오." 등의 **보안 운영 가이드라인을 명시**하는 것으로 충분히 조치 가능합니다.

### 6. 의존성 버전 고정 (Dependency Pinning)
- **적용 가능 여부:** 🟢 **완벽히 적용 가능**
- **기능 영향도:** 없음 (오히려 안정성 향상)
- **조치 방안:** `requirements.txt`에 적힌 라이브러리들을 현재 테스트가 완료된 안전한 특정 버전으로 묶어버립니다. (예: `PyMuPDF==1.24.2`, `pywebview==5.0.5` 등). 이를 통해 향후 빌드 시 악성 업데이트가 섞여 들어오는 것을 막을 수 있습니다.

### 7. 배포 실행파일 검증 정보 제공
- **적용 가능 여부:** 🟢 **완벽히 적용 가능**
- **조치 방안:** 빌드된 `.exe` 파일의 SHA-256 해시값(지문)을 추출하여 깃허브 Release 페이지에 함께 적어줍니다. 다운로드 받는 공공기관 사용자가 해시값을 비교하여 "위변조 되지 않은 원본 프로그램"임을 확신할 수 있게 됩니다.

## 📝 다음 단계 (Next Step)
검토 결과, 보안 전문가의 지적은 매우 타당하며, **현재 코드베이스의 아키텍처를 뒤엎지 않고도 몇 줄의 필터링 함수와 검증 로직 추가만으로 모두 조치할 수 있는 "가성비가 매우 좋은" 보안 패치**입니다.

조치를 원하신다면, 위 7가지 항목을 모두 반영하는 **[보안 패치 실행 계획]**을 세우고 바로 코드 수정을 진행할 수 있습니다.
