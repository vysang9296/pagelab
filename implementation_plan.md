## 🚀 2. 우클릭 내보내기 및 파일명 지정 (Export UX)

### A. 왼쪽 원본 파일 우클릭 시
*   **[원본 다운로드]:** 중앙 뷰어에서 무슨 짓(삭제, 회전, 순서변경)을 했든 전부 무시하고, 최초 업로드된 순수한 원본 상태 그대로 다운로드합니다. (기본 파일명: `원본_파일명`)
*   **[편집 다운로드]:** 중앙 뷰어에서 적용한 변경사항(삭제된 페이지 제외, 회전 적용, 순서 변경)을 모두 반영하여 새로운 PDF로 병합 후 다운로드합니다. (기본 파일명: `편집_파일명`)

### B. 오른쪽 분류 폴더 우클릭 시 (단일 선택)
*   **[통합 다운로드]:** 폴더 내 문서를 하나의 PDF로 병합. (기본 파일명: `폴더명.pdf`)
*   **[파일별 다운로드]:** 폴더 내 문서를 낱장 PDF로 분리 후 1개의 ZIP 압축. (기본 파일명: `폴더명.zip`)

### C. 오른쪽 분류 폴더 우클릭 시 (다중 선택 상태)
*   **[통합 다운로드]:** 각 폴더를 1개의 PDF로 묶은 뒤, 그 PDF들을 하나의 ZIP으로 압축.
*   **[파일별 다운로드]:** 각 폴더를 개별 ZIP(낱장들)으로 묶은 뒤, 그 ZIP 파일들을 다시 최상위 ZIP으로 압축(**이중 압축**).

> **※ 파일명 지정:** 모든 우클릭 다운로드 시 윈도우 기본 저장 창이 열리며, 사용자가 원하는 폴더 경로와 파일명을 직접 지정할 수 있습니다.

## 🛡️ 3. 원본 유지 및 동적 편집 로직 (Core Concept)

선생님께서 강조하신 철학을 코드로 완벽히 구현합니다.

1.  **복사형 할당 (원본 불변의 법칙):**
    *   중앙 뷰어(원본 모드)에서 썸네일을 오른쪽 분류 폴더로 아무리 많이 집어넣어도, 중앙 뷰어의 썸네일은 **절대 사라지지 않고 원형을 유지**합니다.
2.  **동적 삭제/제외 (모드별 차별화):**
    *   **원본 보기 모드:** 삭제 버튼 `[✖]` 클릭 시 썸네일이 **흐려지고(Dim)** 취소선만 그어집니다. 이는 영구 삭제가 아니며, 단순히 `[편집 다운로드]` 시 이 페이지를 뺄 것이라는 '제외 마커' 역할만 합니다.
    *   **분류 폴더 모드:** 삭제 버튼 클릭 시 썸네일이 화면에서 **즉시 완전히 사라집니다**. (해당 분류 폴더에서 문서를 아예 빼버리는 직관적인 동작).
4.  **표준 범위 지정 및 일괄 제어 (Shift & Ctrl):**
    *   OS 표준에 맞춰 `Shift` 키로 대량의 페이지 범위를 한 번에 블록 지정할 수 있습니다. 선택된 다수의 페이지는 상단 툴바를 통해 일괄 회전시키거나, 한 번에 '삭제(제외)'하거나, 통째로 우측 그룹에 끌어다 놓을 수 있습니다.
5.  **상세 라벨링:**
    *   분류 폴더 모드에서 페이지를 확인할 때, 썸네일 하단에 **`p.4 (원본_파일명.pdf)`** 형태로 표출하여 어느 파일에서 발췌해 왔는지 한눈에 알 수 있게 합니다.

## Verification Plan
1. **원본 보존 및 편집 추출 테스트:** 원본 파일의 2, 4페이지를 흐리게(제외) 만들고 순서를 섞은 뒤, `[원본 다운로드]`를 하면 원본 그대로 나오는지, `[편집 다운로드]`를 하면 제외 및 섞인 대로 나오는지 대조 검증.
2. **이중 압축 로직 테스트:** 2개의 폴더 다중 선택 후 '파일별 다운로드' 시 이중 압축(ZIP in ZIP) 검증.
3. **복사형 할당 테스트:** 원본 페이지를 여러 그룹에 넣어도 원본 화면에서는 썸네일이 멀쩡히 남아있는지 검증.
